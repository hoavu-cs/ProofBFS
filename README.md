# proof-bfs
This is my attempt at creating a system for searching for interesting mathematical facts using BFS.
I adopt a BFS-style search to explore the space of mathematical statements, starting from a set of initial statements and iteratively proposing new statements based on the current set of facts. 

To get started, create a file in the `outputs` named `statements_id.txt` where `id` is a unique identifier for your search. Add some initial statements to the file, one per line. `Goal:` can be added at the end of the file to specify a goal statement to prove. The conversation will continue until the goal is reached or the maximum number of rounds is reached. 

At each round, the program will propose a new **incremental** statement based on the current set of facts

```
1. y ≥ 1 
2. x > 0 
3. y ≥ 4xln(x) 

Goal: Show that y ≥ x ln(2y). 
```

Then run the program with `python app.py id` to start the search. 
The program will iteratively propose new incremental statements and check them for validity, logging the results in `outputs/log_id.txt`. 
The statements and proofs are generated by a two-agent conversation loop, where one agent (often the chat-version agent) proposes new statements and the other (often the reasoning agent) checks them for correctness. 
Approved statements and proofs will be added to the fact pool for subsequent rounds.

The new correct statements will be appended to the `outputs/statements_id.txt` file, and the program will continue until it reaches the maximum number of rounds or no new statements can be derived. The conversation log will be saved in `outputs/log_id.txt` for review.
